# Бизнес-требования

## Цель

Подготовить демонстрационный инструмент для независимой проверки скоринговых ML-моделей и LLM-ответов.

## Пользователь

Validation DS / аналитик, который проверяет корректность модели, устойчивость качества и риски применения AI-решений.

## Основные требования

1. Загрузить историческую скоринговую выборку.
2. Обучить baseline и challenger-модели.
3. Посчитать ROC-AUC, Gini, KS, Precision, Recall и F1.
4. Провести backtest по историческим периодам.
5. Проверить drift признаков через PSI.
6. Оценить LLM-ответы по релевантности, фактологичности, полноте и риску галлюцинаций.
7. Сохранить generated CSV-отчёты для UI.
8. Показать результаты в Blazor-интерфейсе.
