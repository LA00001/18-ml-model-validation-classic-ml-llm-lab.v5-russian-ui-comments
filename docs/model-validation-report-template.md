# Шаблон отчёта по валидации ML-модели

## 1. Описание модели

- Название модели
- Бизнес-задача
- Целевая переменная
- Горизонт прогноза
- Используемые признаки

## 2. Метрики качества

- ROC-AUC
- Gini
- KS
- Precision
- Recall
- F1

## 3. Backtest

Проверить качество модели на исторических периодах и выявить деградацию качества.

## 4. PSI / drift monitoring

Проверить сдвиг распределения признаков между train и holdout.

## 5. Вывод validation DS

- Можно ли использовать модель
- Какие есть ограничения
- Нужны ли дополнительные проверки
- Какие метрики поставить на мониторинг
