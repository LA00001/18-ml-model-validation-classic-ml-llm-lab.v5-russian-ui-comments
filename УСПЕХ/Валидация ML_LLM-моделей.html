Выводы validation DS

  • Лучшая модель по ROC-AUC: challenger: градиентный бустинг; валидация должна проверить не только качество, но и стабильность, интерпретируемость и применимость бизнес-правил.
  • PSI/drift требует внимания по признакам: возраст, ежемесячный доход, долговая нагрузка, недавние кредитные запросы; нужен регулярный мониторинг и пороги алертов.
  • Backtest нашёл 3 период(а/ов) со статусом наблюдение/критично; качество модели нужно контролировать по временным срезам.
  • Оценка LLM нашла 2 кейс(а/ов) высокого риска; ответы LLM для отчёта по валидации должны проходить ручную проверку.
  • Для production-подхода нужны карточка модели, отчёт по валидации, проверка challenger-модели, пороги качества, алерты и регламент пересмотра модели.
